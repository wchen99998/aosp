# v0.1.3 Release Notes

Validated `aosp_cf_arm64_only_phone_qemu` build outputs with dual gralloc
backend support (minigbm + ranchu).

## What Changed

- The QEMU product now ships both gralloc backends as vendor APEXes:
  - `com.google.cf.gralloc` (minigbm) — default for plain-QEMU/software-GL
  - `com.google.cf.gralloc.ranchu` (gfxstream-native) — for gfxstream hosts
- Boot-time selection via `QEMU_BOOT_HARDWARE_GRALLOC` picks the active
  backend, which maps to `androidboot.vendor.apex.com.google.cf.gralloc`.
- SELinux file_contexts added for the ranchu allocator binary and mapper.
- Launch scripts (`run_plain_qemu.sh`, `prepare_plain_qemu.sh`) updated with
  `QEMU_BOOT_VENDOR_APEX_GRALLOC` derivation from `QEMU_BOOT_HARDWARE_GRALLOC`.

## What Is Included

- `aosp_cf_arm64_only_phone_qemu-out-2026-03-21.tar.zst`
- matching manifest and SHA-256 files

## How To Use It

1. Extract the archive under the repository root.
2. Place the included image files under `src/out/target/product/vsoc_arm64_only`.
3. Run `./scripts/prepare_plain_qemu.sh`.
4. Follow `docs/BOOT.md` for the validated no-harness QEMU launch flow.

For gfxstream hosts, set `QEMU_BOOT_HARDWARE_GRALLOC=ranchu` in the launch
environment.
