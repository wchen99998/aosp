# v0.1.3 Release Notes

Validated `aosp_cf_arm64_only_phone_qemu` build outputs with dual gralloc
backend support (minigbm + ranchu).

## What Changed

- The QEMU product now ships both gralloc backends as standalone vendor
  binaries (not APEXes):
  - `android.hardware.graphics.allocator-service.minigbm` + `mapper.minigbm`
    — default for plain-QEMU/software-GL
  - `android.hardware.graphics.allocator-service.ranchu` + `mapper.ranchu`
    — gfxstream-native, for gfxstream-capable hosts
- The QEMU init fragment (`init.vendor.qemu.rc`) stops the default minigbm
  allocator and starts the ranchu one when `ro.hardware.gralloc=ranchu`.
- SELinux file_contexts added for the ranchu allocator binary and mapper.
- The goldfish `allocator-service.ranchu` module no longer bundles init_rc or
  vintf_fragments so it coexists with the minigbm allocator on Cuttlefish.

## What Is Included

- `aosp_cf_arm64_only_phone_qemu-out-2026-03-21.tar.zst`
- matching manifest and SHA-256 files

## How To Use It

1. Extract the archive under the repository root.
2. Place the included image files under `src/out/target/product/vsoc_arm64_only`.
3. Run `./scripts/prepare_plain_qemu.sh`.
4. Follow `docs/BOOT.md` for the validated no-harness QEMU launch flow.

For gfxstream hosts, set `QEMU_BOOT_HARDWARE_GRALLOC=ranchu` in the launch
environment.
