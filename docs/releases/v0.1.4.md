# v0.1.4 Release Notes

Rebuilt `aosp_cf_arm64_only_phone_qemu` outputs after fixing the QEMU gralloc
selector to key off `ro.boot.hardware.gralloc` during `early-init`.

## What Changed

- `init.vendor.qemu.rc` now selects the ranchu allocator from the boot property
  instead of the later-derived `ro.hardware.gralloc` property.
- This avoids the `early-init` ordering bug where `allocator.rc` could still
  leave the default minigbm service selected even when
  `QEMU_BOOT_HARDWARE_GRALLOC=ranchu`.
- The main graphics-contract and validation docs were updated to describe the
  boot-property-based selector.

## What Is Included

- `aosp_cf_arm64_only_phone_qemu-out-2026-03-21-r1.tar.zst`
- matching manifest and SHA-256 files
- generated release notes under `release-assets/v0.1.4-notes.md`

## How To Use It

1. Extract the archive under the repository root.
2. Place the included image files under `src/out/target/product/vsoc_arm64_only`.
3. Run `./scripts/prepare_plain_qemu.sh`.
4. Follow `docs/BOOT.md` for the plain-QEMU launch flow.
