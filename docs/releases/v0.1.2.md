# v0.1.2 Release Notes

Validated `aosp_cf_arm64_only_phone_qemu` build outputs for the 2026-03-21
guest-refresh-policy update.

## What Is Included

- `aosp_cf_arm64_only_phone_qemu-out-2026-03-21.tar.zst`
- matching manifest and SHA-256 files

## Why This Asset Format

- the direct processed `plain-qemu/os-disk.raw` boot image is too large to
  publish comfortably
- this release instead ships the compiled output set that was used to produce
  the validated plain-QEMU bundle

## How To Use It

1. Extract the archive under the repository root.
2. Place the included image files under `src/out/target/product/vsoc_arm64_only`.
3. Run `./scripts/prepare_plain_qemu.sh`.
4. Follow `docs/BOOT.md` for the validated no-harness QEMU launch flow.
