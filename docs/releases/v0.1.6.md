# v0.1.6 Release Notes

Rebuilt `aosp_cf_arm64_only_phone_qemu` outputs after adding guest-side device
permissions for the goldfish address-space and pipe devices required by the
ranchu gralloc allocator.

## What Changed

- Added ueventd entries for `/dev/goldfish_address_space` and
  `/dev/goldfish_pipe_dprctd` (0666 system:system) to the shared cuttlefish
  `ueventd.rc`.  These are no-ops on non-QEMU cuttlefish builds because the
  device nodes are never created.
- Created a QEMU-specific SELinux policy directory
  (`shared/config/qemu/sepolicy/`) containing:
  - `device.te` — defines the `qemu_device` SELinux type
  - `domain.te` — grants domain-wide access to `qemu_device` character devices
    (required because gfxstream same-process HAL libraries load into arbitrary
    app domains)
  - `file_contexts` — labels `/dev/goldfish_address_space` and
    `/dev/goldfish_pipe_dprctd` as `qemu_device`
- Wired the new sepolicy directory into the QEMU product makefile via
  `BOARD_VENDOR_SEPOLICY_DIRS`, so only the QEMU product carries these policies.

## Why

The ranchu gralloc allocator opens `/dev/goldfish_address_space` (via
`GoldfishAddressSpaceHostMemoryAllocator`) and `/dev/goldfish_pipe_dprctd` (via
`qemu_pipe_open("refcount")`).  Without ueventd rules the kernel-created nodes
default to `root:root 0600`, and without SELinux labels they are assigned the
`unlabeled` type — both of which block the allocator from opening them, causing
a fatal crash at boot.

## Validation

- Full plain-QEMU software-GL validation with `QEMU_BOOT_HARDWARE_GRALLOC=minigbm`:
  - `adb -s 127.0.0.1:6570 get-state` returned `device`
  - `adb -s 127.0.0.1:6570 shell getprop sys.boot_completed` returned `1`
  - `adb -s 127.0.0.1:6570 shell getprop ro.hardware.gralloc` returned
    `minigbm`
  - `adb -s 127.0.0.1:6570 shell ps -A | grep allocator` showed
    `android.hardware.graphics.allocator-service.minigbm`
- Confirmed `vendor/etc/ueventd.rc` in the built image contains the goldfish
  entries (`/dev/goldfish_address_space` and `/dev/goldfish_pipe_dprctd`).
- Confirmed `vendor/etc/selinux/vendor_file_contexts` labels both goldfish
  device nodes as `qemu_device`.

## How To Use It

1. Extract the archive under the repository root.
2. Place the included image files under `src/out/target/product/vsoc_arm64_only`.
3. Run `./scripts/prepare_plain_qemu.sh`.
4. Follow `docs/BOOT.md` for the plain-QEMU launch flow.
